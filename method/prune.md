# Method: Prune / iterate (step 12)

_The counterweight to compound: keep the canonical context lean, current, and non-contradictory.
Context that only grows rots. Drafted 2026-06-30._

## Goal
A Brief and Voice Profile that stay sharp and trustworthy — not an ever-growing pile.

## Trigger
Periodically: every N sessions, when a file crosses its **soft size budget**, or on a **pivot**.

## Process
1. **Dedupe & merge** overlapping rules/entries.
2. **Resolve drift / era** (Q4): down-weight or retire old-era signal; on contradictions, newer
   practice wins — or ask the writer.
3. **Drop the stale:** rules that no longer fire, objectives met or abandoned, tensions resolved.
4. **Pivot handling:** when the newsletter changes direction, edit the **Brief first**, deliberately
   (it's the source of truth) — then the work follows. Re-confirm with the writer.
5. **Trim the changelog** itself so it stays a useful history, not a log dump.

## Budget
Each context file has a soft size budget (it's a *working reference*, not an archive). If exceeded,
**prune before compounding more**.

## Output
Leaner `newsletter-brief.md` / `voice-profile.md`; bumped "Last confirmed" date; a clean changelog.
