# Method: Compound (step 11)

_At session end, turn what was learned into durable, approved updates to the canonical context —
so context improves with use, with near-zero authoring burden. Drafted 2026-06-30._

## Goal
Make the Brief and Voice Profile get *better* every time the writer uses the kit — without asking
them to maintain anything.

## Trigger
End of a session that touched the newsletter. Ideally **auto-prompted** (a session-end hook);
minimally a skill the writer runs.

## Process
1. **Scan the session for durable signal:** corrections the writer made, "I'd never say that"
   moments, a recurring new topic/motif, a voice rule that fired repeatedly, a drift or pivot, a
   new boundary, a shifted objective.
2. **Filter to the key essential few (3–5 max).** Only what **generalizes** beyond this one piece.
   A one-off does not earn a context entry (load + anti-bloat discipline).
3. **Route each to its home + section:**
   - subject · topics · objectives · tensions · vocabulary → **the Brief**
   - voice rules · "never" list · naturalness → **the Voice Profile**
4. **Propose as a short list; the writer approves / edits / rejects each** (the gate). Never silent.
5. **Apply approved edits IN PLACE** to the canonical file, routed to the right section, and append a
   one-line **changelog** entry (date · what · why · session).

## Anti-bloat rule
Prefer **sharpening an existing rule** to adding a new one. If two entries overlap, merge. If it
won't recur, don't add it. (Compounding without this just bloats — that's what `prune` cleans.)

## Output
Updated `newsletter-brief.md` / `voice-profile.md` + changelog entries. Bump "Last confirmed" if the
identity changed.
