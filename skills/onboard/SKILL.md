# Onboard — "these are my tools, how I work today" (step 0)

The first moment: organize **where the writer's work lives**. This is about orientation
and connection, not extraction. It does **not** fetch the corpus.

## What it captures
- **Tools & workflow today** — how they actually work now (e.g. a reading database, a
  daily source feed, where they draft, where they publish). "These are my tools."
- **Knowledge homes** — connect/declare the sources: Substack URL (required, v1), and
  Notion / Drive / notes for later phases.
- Confirm the Substack URL resolves (light check only — no archive pull).

## What it does NOT do
- ❌ No corpus fetch / reading. The heavy read is the **last** step of Phase A and is
  gated behind describe (1) and stated identity (2). Onboard never triggers it.

## Output
`workspace.md` — the source map + how-they-work notes. Pointers other steps will use.

## The Phase A sequence (light → heavy; this skill is only the first beat)
1. **Onboard (this skill)** — where work lives.
2. **Describe** (the describe step) — "how I see my newsletter" → `newsletter-context/seed-context.md`.
3. **Stated identity** (the `pull-identity` skill) — "what it says about itself" → `newsletter-context/identity-stated.md`.
4. **Read it all** (`extract-corpus` skill) — the agent fetches + reads the full corpus. **Last.**

Then, and only then → **Milestone 1** (distill subject → reconcile → Newsletter Brief).

End by pointing the writer to step 2 (describe). Do not run later steps yourself.

---
_**Language:** if the newsletter isn't in English, converse with the writer — and write outputs — in the newsletter's language._
