# Reconcile (step 5) — the engine

Turn three static views into directional context. The corpus is descriptive (what the newsletter
*was*); this dialogue is the only place **aspiration** (what it *wants to be*) enters.

## Precondition
`newsletter-context/seed-context.md` (said), `newsletter-context/identity-stated.md` (stated), `newsletter-context/subject-observed.md` (shown).

## Procedure
1. **Three-way diff.** Lay the views side by side; find disagreements. Four gap types: said-not-shown · shown-not-said · stated-vs-shown · said-vs-stated.
2. **Prioritize** the top 3–5 gaps that most change the Brief (overwhelm kills the gate).
3. **Ask, don't assert** (Q7 — anchoring). One *open* question per gap — never leading. Present the agent's read as challengeable, not as the answer.
4. **Capture + classify** each answer: aspiration (→ objectives) · drift (→ down-weight/prune) · latent strength (→ surface) · noise (→ exclude).

## Note
This step is **interactive** — the writer's answers are the whole point. Do not resolve gaps for
them. If they skip one, mark it open.

## Output
`newsletter-context/reconcile.md` — gaps, questions, and resolutions. → hand to **`brief`** (step 6).

---
_**Language:** if the newsletter isn't in English, converse with the writer — and write outputs — in the newsletter's language._
